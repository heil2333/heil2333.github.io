服务器信息

Minecraft对服务器的要求：
CPU：2+

内存 容量(人)
2GB 20-40
3GB 30-60
8GB+ 60+

首先，你得先有一台的服务器。
这里我用我这个比较远古的Synology DS216J举例，
下面是服务器的参数。

DS216J 数值
CPU时钟频率 1GHz
CPU内核数 2
内存 512 MB

与服务器建立一个链接

服务器设置端口

此外你还需要一个admin的账户，用于登录上你的服务器网站。

需要去应用商店中下载一个JAVA并安装。

在应用列表里找到控制面板
打开应用程序下的终端机和SNMP
点击启用SSH功能
设置端口，我这设置了一个最不常用的端口 1464
最后点击保存就可以了。

环境变量

这里要说一说环境变量。
右键开始徽标
选择系统
在打开的窗口中选择关于
在页面最下面相关设置
选择系统信息
新窗口中选择高级系统设置（管理员）
系统属性里的高级的最后一个选项环境变量。

环境变量分为两个，
一个是用户变量，另一个是系统变量。
在用户变量里创建的内容会同样在系统变量中创建，包括变量的值。

我们需要修改的是用户变量Path的值,选中并编辑新建一个值, 把刚才安装的<OpenSSH的安装路径>\bin\添加进来确定保存即可。
Win+R打开运行框,输入cmd按回车启动命令与提示符。
输入ssh并回车。

部署你的Minecraft！

下载和安装服务端

下载mc服务器包:
wget https://s3.amazonaws.com/Minecraft.Download/versions/<你要下载的版本号>/minecraft_server.<你要下载的版本号>.jar

java -Xms[初始启动分配内存] -Xmx[最大分配内存] -jar [jar包所在路径]/minecraft_server.[版本号].jar nogui

关于命令最后一个参数 nogui，是分开的单词 no gui，意思是不需要图形界面，这样会大大减小内存的使用，如果你的Linux是有图形界面的，就可以不打这个 nogui。

非正版登录

若不是正版账号的话是无法登录服务器的，
更改Minecraft下的server.properties文件。

server.properties点击查看相关设置文档
找到并更改以下内容
online-mode=false

保持我们的MC服务会话

如果你不是Sylonogy的服务器你可以尝试以下项目
安装 GNU Screen
screen -AmRvx -ls -wipe-h <行数>-s

参数说明
-A 　将所有的视窗都调整为目前终端机的大小。
-d <作业名称> 　将指定的screen作业离线。
-h <行数> 　指定视窗的缓冲区行数。
-m 　即使目前已在作业中的screen作业，仍强制建立新的screen作业。
-r <作业名称> 　恢复离线的screen作业。
-R 　先试图恢复离线的作业。若找不到离线的作业，即建立新的screen作业。
-s 　指定建立新视窗时，所要执行的shell。
-S <作业名称> 　指定screen作业的名称。
-v 　显示版本信息。
-x 　恢复之前离线的screen作业。
-ls或–list 　显示目前所有的screen作业。
-wipe 　检查目前所有的screen作业，并删除已经无法使用的screen作业。
常用screen参数
screen -S yourname -> 新建一个叫yourname的session
screen -ls -> 列出当前所有的session
screen -r yourname -> 回到yourname这个session
screen -d yourname -> 远程detach某个session
screen -d -r yourname -> 结束当前session并回到yourname这个session

新建一个screen会话
screen -S mcs
